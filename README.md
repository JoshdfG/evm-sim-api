# EVM Transaction Simulation API

Simulate any EVM transaction before it's sent — token deltas, gas usage, revert reasons, and risk flags. Like Tenderly's simulate API, self-hosted in Go.

Built on the [go-production-starter](https://github.com/JoshdfG/go-production-starter) template: Clean Architecture, Gin, zerolog, Redis, sqlc, PostgreSQL.

---

## Stack

| Concern | Tool |
|---|---|
| HTTP framework | Gin |
| EVM execution | go-ethereum (`ethclient` + `debug_traceCall`) |
| Archive RPC | Alchemy / Infura / QuickNode |
| Database | PostgreSQL 16 via pgx v5 + sqlc |
| Cache | Redis 7 — finalized sims cached forever by content hash |
| Logging | zerolog (structured JSON) |
| API auth | `X-API-Key` header (B2B model) |
| Rate limiting | ulule/limiter (per-IP, in-memory) |
| API docs | Swagger via swaggo |
| Config | cleanenv (.env + env vars, fail-fast on missing required) |
| CI | GitHub Actions |

## Architecture

The dependency rule: **inner layers never import outer layers**.

```
HTTP Handler  (controller/http)
     │ calls
     ▼
SimulationUseCase  (usecase)          ← pure Go, no infrastructure imports
     │ calls via interface
     ▼
EVMFork interface
     │ implemented by:
     ▼
LoggingFork → CachingFork → ArchiveNodeFork   (repo/evm)
                                  │
                             go-ethereum
                         eth_call + debug_traceCall
```

The decorator chain is assembled once in `main.go`:

```go
archiveFork := evm.NewArchiveNodeFork(cfg.EVM.ArchiveNodeURL, ...)
cachedFork  := repo.NewSimulationCacheRepo(archiveFork, rdb, ...)
loggedFork  := repo.NewSimulationLoggingRepo(cachedFork, log)
simUC       := usecase.NewSimulationUseCase(loggedFork, simRepo, tokenDecoder, riskAnalyzer)
```

To add Prometheus metrics: write a `MetricsFork` that wraps `loggedFork`. Zero changes to existing code.

## Quick start

```bash
# 1. Clone and configure
git clone https://github.com/joshdfg/evm-sim-api
cd evm-sim-api
cp .env.example .env
# Edit .env — set EVM_ARCHIVE_NODE_URL to your Alchemy/Infura key

# 2. Start infrastructure
make infra

# 3. Apply migrations
make migrate

# 4. Seed a local dev API key
make seed
# Output: Dev API key: dev-test-key-00000000

# 5. Run
make run
```

Swagger UI: http://localhost:8080/swagger/index.html

## Risk flags

| Code | Severity | Trigger |
|---|---|---|
| `UNLIMITED_APPROVAL` | critical | ERC20 `approve(spender, MAX_UINT256)` |
| `SET_APPROVAL_FOR_ALL` | critical | NFT `setApprovalForAll(operator, true)` |
| `SELFDESTRUCT` | critical | Contract executes `SELFDESTRUCT` |
| `HIGH_NATIVE_TRANSFER` | warning | Sends ≥ 1 ETH from the `from` address |
| `UNEXPECTED_DELEGATECALL` | warning | `DELEGATECALL` appears in the trace |
| `REENTRANCY_PATTERN` | warning | Same contract address appears > 1 time in trace |

Adding a new rule: add a private method to `DefaultRiskAnalyzer` and call it from `Analyze()`. Write a test. Done.

## Configuration

| Variable | Required | Default | Description |
|---|---|---|---|
| `EVM_ARCHIVE_NODE_URL` | ✅ | — | Archive RPC (Alchemy/Infura/QuickNode) |
| `EVM_SIM_TIMEOUT_SECONDS` | | `30` | Max seconds per simulation |
| `EVM_MAX_GAS_LIMIT` | | `30000000` | Gas cap when caller sends 0 |
| `EVM_CACHE_TTL_SECONDS` | | `0` | Redis TTL for finalized blocks (0 = forever) |
| `EVM_PENDING_CACHE_TTL_SECONDS` | | `15` | Redis TTL for latest-block sims |
| `POSTGRES_USER/PASSWORD/DB` | ✅ | — | Postgres credentials |
| `REDIS_HOST/PORT` | | `localhost:6379` | Redis connection |
| `LOG_LEVEL` | | `info` | `debug` \| `info` \| `warn` \| `error` |

## Running tests

```bash
# Unit tests — no Postgres, no Redis, no archive node
make test

# Verbose
go test ./internal/usecase/... -v -race
```

All business logic is testable with zero infrastructure because the usecase layer depends only on interfaces, and the tests supply mock implementations.
